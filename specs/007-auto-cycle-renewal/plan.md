# Implementation Plan: Automatic Monthly Cycle Renewal

**Branch**: `007-auto-cycle-renewal` | **Date**: 2026-06-30 | **Spec**: [spec.md](spec.md)

## Summary

Replace the manual "reset cycle" button with an automatic cycle detection system. On every app launch, the app checks whether the user's salary day has arrived and no cycle for this month exists yet. If so, a new cycle is created automatically and the user is routed to the salary split screen before accessing any other feature. Debts, lendings, and goals persist unchanged across cycles. The salary day change rule is enforced by updating only `user.salaryDay` — the active cycle's dates are never retroactively modified.

## Detailed Implementation

### 1. `CycleRepository` interface

Add to `cycle_repository.dart`:
```dart
Future<FinancialCycleEntity?> getCycleForMonth(int year, int month);
```

### 2. `CyclesDao` — new query

```dart
Future<FinancialCycleRow?> getCycleForMonth(int year, int month) {
  final startOfMonth = DateTime(year, month, 1);
  final endOfMonth = DateTime(year, month + 1, 1);
  return (select(financialCycles)
    ..where((t) =>
        t.startDate.isBiggerOrEqualValue(startOfMonth) &
        t.startDate.isSmallerThanValue(endOfMonth))
    ..limit(1))
    .getSingleOrNull();
}
```

### 3. `CheckAndStartCycleUseCase`

```dart
// Returns new FinancialCycleEntity if a new cycle was created (needs salary split).
// Returns null if no action needed.
Future<FinancialCycleEntity?> call() async {
  final user = await _userRepo.getUser();
  if (user == null) return null;

  final now = DateTime.now();
  final today = DateTime(now.year, now.month, now.day);
  final salaryDay = user.salaryDay;

  // Clamp salary day to month's actual days
  final daysInThisMonth = DateTime(today.year, today.month + 1, 0).day;
  final effectiveDay = salaryDay > daysInThisMonth ? daysInThisMonth : salaryDay;
  final thisSalaryDate = DateTime(today.year, today.month, effectiveDay);

  // Not yet salary day
  if (today.isBefore(thisSalaryDate)) return null;

  // Already have a cycle for this month
  final existing = await _cycleRepo.getCycleForMonth(today.year, today.month);
  if (existing != null) return null;

  // Close current active cycle (if any), deposit remaining balance to savings
  final active = await _cycleRepo.getActiveCycle();
  if (active != null) {
    await _depositSavings(active.id);
    await _cycleRepo.closeCycle(active.id);
  }

  // Compute new cycle end date
  final nextMonth = today.month + 1;
  final nextYear = nextMonth > 12 ? today.year + 1 : today.year;
  final normalizedNextMonth = nextMonth > 12 ? 1 : nextMonth;
  final daysInNextMonth = DateTime(nextYear, normalizedNextMonth + 1, 0).day;
  final nextEffectiveDay = salaryDay > daysInNextMonth ? daysInNextMonth : salaryDay;
  final nextSalaryDate = DateTime(nextYear, normalizedNextMonth, nextEffectiveDay);
  final cycleEnd = nextSalaryDate.subtract(const Duration(days: 1));

  return _cycleRepo.createCycle(
    startDate: thisSalaryDate,
    endDate: cycleEnd,
    salaryAmount: user.monthlySalary,
  );
}
```

### 4. `AppRouter` — redirect update

After the existing activated-user check, add:
```dart
// Cycle gate: check if a new cycle should start
if (path != '/salary-split' && !path.startsWith('/onboarding')) {
  final newCycle = await Injection.checkAndStartCycleUseCase();

    Injection.pendingCycleForSplit = newCycle;

  }
}
```

### 5. `SalarySplitPage` route builder

```dart
GoRoute(
  path: '/salary-split',
  builder: (context, state) {
    final extra = state.extra as Map<String, dynamic>?;
    final pending = Injection.pendingCycleForSplit;
    Injection.pendingCycleForSplit = null; // consume

    final cycleId = extra?['cycleId'] ?? pending?.id ?? 0;
    final salaryAmount = extra?['salaryAmount'] ?? pending?.salaryAmount ?? 0;
    final onComplete = extra?['onComplete'] as VoidCallback? 
        ?? () => context.go('/home');

    return BlocProvider(
      create: (_) => SalarySplitCubit(...)..initialize(cycleId, salaryAmount),
      child: SalarySplitPage(onComplete: onComplete),
    );
  },
),
```
